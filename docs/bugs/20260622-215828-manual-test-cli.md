## Bug 1: JAVA_HOME 被系统环境变量覆盖导致 UnsupportedClassVersionError

**严重程度**: P1 (阻断)

**现象**: 执行 `bin\code-agent-java.cmd` 报错：
```
Error: A JNI error has occurred, please check your installation and try again
Exception in thread "main" java.lang.UnsupportedClassVersionError: ... has been compiled 
by a more recent version of the Java Runtime (class file version 61.0), this version of 
the Java Runtime only recognizes class file versions up to 52.0
```

**根因**: `bin/code-agent-java.cmd` 第 4 行使用了 `if not defined JAVA_HOME` 条件判断。系统环境变量已设置 `JAVA_HOME=D:\DevTools\JDK\jdk8`（Java 8），导致脚本跳过 Java 25 路径的赋值，使用了 Java 8 运行 Spring Boot 3.x JAR。

**修复**: 将条件赋值改为无条件强制设置：
```diff
- if not defined JAVA_HOME set JAVA_HOME=C:\Users\EDY\.jdks\openjdk-25.0.1
+ REM Force Java 25 to avoid stale JAVA_HOME from environment
+ set JAVA_HOME=C:\Users\EDY\.jdks\openjdk-25.0.1
```

**状态**: ✅ 已修复

## Bug 2: 配置未生效——JAR 构建时间早于配置文件修改时间

**严重程度**: P2 (功能)

**现象**: `application.properties` 中配置了 `codeagent.llm.model=openai:deepseek-v4-flash`，但运行时报错显示使用了默认值 `openai:gpt-4.1`。

**根因**: JAR 文件构建时间（20:36）早于 `application.properties` 和 `application-local.properties` 的修改时间（21:50）。配置修改后未执行 `mvn package`，fat JAR 中打包的是旧版配置文件。

**修复**: 执行 `mvn clean package -DskipTests` 重新构建。

**预防措施**: 任何 `src/main/resources/` 下的配置修改后，必须重新打包才能生效。可考虑后续添加 DevTools 或开发模式下使用 `mvn spring-boot:run` 直接运行以省去打包步骤。

**状态**: ✅ 已修复

## Bug 3: AgentScope ModelRegistry 未集成应用配置——API Key 传递断层

**严重程度**: P0 (阻断)

**现象**: 配置生效后（模型正确识别为 `openai:deepseek-v4-flash`），仍然报错：
```
Caused by: java.lang.IllegalStateException: Environment variable OPENAI_API_KEY is 
required to auto-create model: openai:deepseek-v4-flash
```

**根因**: `AgentScopeOrchestrator.buildAgent()` 调用 `HarnessAgent.Builder.model(modelId)` 时，AgentScope 内部通过 `ModelRegistry.resolve()` 自动创建模型实例。该注册表直接从环境变量 `OPENAI_API_KEY` 读取 API Key，完全绕过了应用的 `LlmProperties.openai.apiKey` 和 `LlmProperties.openai.baseUrl` 配置。

数据流断裂：
```
application-local.properties
  → codeagent.llm.openai.api-key=sk-xxx
  → codeagent.llm.openai.base-url=https://api.deepseek.com
  → LlmProperties (Spring @ConfigurationProperties)
  → ❌ 未传递给 AgentScope ModelRegistry
  → ❌ AgentScope 自行读取 OPENAI_API_KEY 环境变量
```

**影响范围**: 所有 LLM 调用（chat / generate / explain）均无法正常工作。

**修复方向**: 在 `AgentScopeOrchestrator.buildAgent()` 中使用 `OpenAIChatModel.builder()` 编程式构建模型实例，直接传入 `apiKey` 和 `baseUrl`，然后通过 `HarnessAgent.builder().model(Model)`（而非字符串）注入，彻底绕过 `ModelRegistry` 的环境变量依赖。

```diff
- .model(model)  // 字符串 → 触发 ModelRegistry 读 OPENAI_API_KEY
+ .model(createModel(modelId, llmProperties))  // Model 实例 → 使用配置属性
```

**状态**: ✅ 已修复（详见 `AgentScopeOrchestrator.java:64-118`）

## Bug 4: spring.shell.interactive.enabled=true 导致命令行参数被忽略

**严重程度**: P3 (体验)

**现象**: 执行 `bin\code-agent-java.cmd chat --prompt "hello"` 时，参数被忽略，进入交互式 shell 等待输入。
```
WARN ... Running in interactive mode, arguments will be ignored
```

**根因**: `application.properties` 第 3 行 `spring.shell.interactive.enabled=true`，Spring Shell 启动交互式 REPL 而非解析命令行参数。

**修复**:
- 交互式使用：保持 `true`（当前默认）
- 非交互式使用（脚本/流水线）：`bin\code-agent-java.cmd chat --prompt "hello" --spring.shell.interactive.enabled=false`

**建议**: 考虑将默认值改为 `false`，通过命令行参数 `--spring.shell.interactive.enabled=true` 启用交互模式，使非交互模式为默认行为（与 `docs/tasks.md` 中记录的原始设计一致）。

**状态**: ⚠ 设计问题，待决策
